Quick Start

Get started with SoundByte by running your first experiment. This guide covers the basic workflow and essential concepts.

1. Initialize a Configuration

Start by creating a sample configuration file for a classification task:

soundbyte init --template classification

This creates a config.json file with a basic classification setup.

2. Run Your First Experiment

Execute the experiment using the generated configuration:

soundbyte run config.json

The toolkit will automatically handle dataset loading, model creation, training, and evaluation.

3. Validate Configuration

Before running experiments, validate your configuration:

soundbyte validate config.json

This checks for configuration errors and ensures all components are available.

Command Line Interface

SoundByte provides a comprehensive CLI for managing experiments, validating configurations, and exploring components.

soundbyte run

Execute experiments based on configuration files.

soundbyte run <config.json>

soundbyte validate

Validate configuration files before execution.

soundbyte validate <config.json>

soundbyte list-components

Display all available components in the toolkit.

soundbyte list-components

Programmatic API

Use SoundByte components programmatically for advanced integration scenarios.

Basic Usage

from soundbyte import SoundByteRunner from soundbyte.config import ExperimentConfig # Load configuration config = ExperimentConfig.from_json("config.json") # Create runner runner = SoundByteRunner(config) # Execute experiment results = runner.run() print(f"Final accuracy: {results.accuracy}")

Component Access

from soundbyte.registry import ComponentRegistry # Get available components registry = ComponentRegistry() models = registry.get_models() # Create specific components model = registry.create_model("resnet18", num_classes=10)
